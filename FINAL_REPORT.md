## 2. Introduction

LectureMate는 강의 녹음 파일을 활용해 학습 노트를 효율적으로 정리할 수 있도록 만든 데스크톱 애플리케이션이다. 사용자는 강의별로 녹음 파일을 첨부하거나 직접 녹음할 수 있으며, OpenAI 음성 인식 API를 통해 녹음 내용을 텍스트로 변환할 수 있다. 변환된 원문 텍스트는 다시 OpenAI 텍스트 생성 API를 통해 자동 요약되며, 사용자는 이 결과를 직접 수정하여 최종 정리 노트로 저장할 수 있다.

또한 강의별 복습 일정, 과제 일정, 시험 일정을 등록할 수 있는 일정 관리 기능을 제공한다. 일정은 월간 캘린더와 리스트 형태로 확인할 수 있고, 완료된 일정은 목록에서 숨겨지도록 구현했다. 추가로 현재 열려 있는 노트 내용을 기반으로 AI에게 질문할 수 있는 대화 기능도 포함했다.

이 애플리케이션의 목표는 강의 녹음, 원문 변환, 요약, 정리 노트 작성, 복습 일정 관리를 하나의 흐름으로 연결하여 학습자의 반복적인 정리 작업을 줄이는 것이다.

## 3. Application Functionalities

### 3.1 강의 관리 기능

LectureMate는 강의 단위로 학습 자료를 관리한다. 사용자는 강의 제목과 과목명을 입력하여 새 강의를 생성할 수 있고, 저장된 강의 목록에서 원하는 강의를 선택할 수 있다. 선택한 강의의 제목, 과목명, 생성일, 원문 텍스트, 정리 노트, 일정 정보가 함께 불러와진다.

구현된 기능은 다음과 같다.

- 강의 생성
- 강의 목록 조회
- 강의 선택
- 강의 제목 및 과목명 수정
- 강의 삭제
- 강의별 원문 텍스트와 정리 노트 저장
- 강의별 첨부 녹음 파일 정보 저장

### 3.2 녹음 파일 첨부 및 직접 녹음 기능

사용자는 `mp3`, `wav`, `m4a` 형식의 녹음 파일을 선택할 수 있다. 파일 선택은 JavaFX의 `FileChooser`를 사용하여 구현했다. 파일을 선택하면 파일명과 경로가 화면에 표시되며, 파일 크기가 25MB를 초과할 경우 경고 메시지가 표시된다.

또한 앱 내부에서 직접 녹음하는 기능도 제공한다. 사용자는 녹음을 시작하고, 일시정지하거나 다시 시작할 수 있으며, 녹음을 중지하면 생성된 WAV 파일이 현재 강의에 첨부된다.

### 3.3 음성 텍스트 변환 기능

LectureMate는 OpenAI Audio Transcriptions API를 호출하는 구조를 제공한다. 사용자는 음성 인식 모델을 선택할 수 있으며, 기본 모델은 `gpt-4o-mini-transcribe`이다. 선택 가능한 모델은 다음과 같다.

- `gpt-4o-mini-transcribe`
- `gpt-4o-transcribe`
- `whisper-1`

녹음 언어도 선택할 수 있다.

- 한국어
- 영어
- 일본어
- 중국어
- 자동 감지

API Key는 코드에 직접 작성하지 않고, 환경 변수 `OPENAI_API_KEY`에서 읽도록 구현했다. API Key가 없거나 API 호출이 실패해도 앱이 종료되지 않고 샘플 원문 텍스트를 표시하는 fallback 로직을 적용했다.

### 3.4 자동 요약 및 정리 노트 기능

음성 변환으로 생성된 원문 텍스트를 기반으로 자동 요약을 생성할 수 있다. 요약 기능은 OpenAI Responses API를 사용하는 구조로 구현했다. 사용자는 요약 형식을 선택할 수 있다.

- 기본 강의 요약
- 시험 대비 요약
- 키워드 중심 요약
- 복습 질문 포함 요약

요약 결과는 오른쪽 정리 노트 편집 영역에 표시된다. 사용자는 자동 생성된 요약을 직접 수정할 수 있으며, 수정된 내용은 최종 정리 노트로 저장된다. 정리 노트는 글자 크기와 폰트를 변경할 수 있고, 변경한 스타일도 함께 저장되도록 구현했다.

### 3.5 강의 목록 및 통합 필기 화면

강의 목록 탭에서는 저장된 강의의 원문 텍스트 목록과 최종 정리 노트 목록을 확인할 수 있다. 강의를 선택하면 통합 필기 화면에서 원문 또는 정리 노트를 크게 볼 수 있고, 직접 편집할 수 있다.

또한 강의에 연결된 녹음 파일을 재생할 수 있는 미니 오디오 플레이어를 제공한다. 재생, 일시정지, 정지 버튼과 재생 위치를 조절할 수 있는 슬라이더가 포함되어 있다.

### 3.6 일정 관리 기능

LectureMate는 강의별 일정 관리 기능을 제공한다. 일정 유형은 다음과 같다.

- 복습
- 과제
- 시험
- 기타

일정에는 제목, 설명, 날짜, 시간, 유형, 완료 여부가 저장된다. 일정 관리 탭에서는 월간 캘린더를 제공하며, 미완료 일정이 있는 날짜는 강조 표시된다. 특정 날짜를 선택하면 해당 날짜의 미완료 일정만 필터링해서 볼 수 있고, 전체 일정 보기 버튼을 통해 날짜 필터를 해제할 수 있다.

완료 체크박스를 누르면 일정은 삭제되지 않고 `is_done = 1`로 저장된다. 완료된 일정은 기본 일정 목록과 캘린더 강조 표시에서 사라지도록 구현했다.

### 3.7 AI 질문하기 기능

질문하기 탭에서는 현재 열려 있는 노트 내용을 기반으로 AI에게 질문할 수 있다. 사용자가 질문을 입력하면 앱은 현재 선택된 강의의 원문 텍스트 또는 정리 노트를 컨텍스트로 사용하여 OpenAI API에 요청한다.

API 호출이 실패하거나 API Key가 없는 경우에는 샘플 답변을 표시하여 앱 기능을 시연할 수 있도록 했다. 대화 화면은 사용자 메시지와 AI 답변이 구분되는 말풍선 형태로 구성했다.

## 4. Database Design

LectureMate는 SQLite를 사용하여 데이터를 저장한다. 앱 실행 시 데이터베이스 파일이 없으면 자동으로 생성된다. 주요 테이블은 다음과 같다.

### 4.1 lectures 테이블

강의 정보를 저장하는 테이블이다.

- `id`: 강의 ID
- `title`: 강의 제목
- `subject`: 과목명
- `audio_path`: 첨부된 녹음 파일 경로
- `transcript`: 음성 변환 원문 텍스트
- `summary`: 최종 정리 노트
- `summary_style`: 정리 노트 글자 크기 및 폰트 스타일 정보
- `created_at`: 생성일
- `updated_at`: 수정일

### 4.2 schedules 테이블

강의별 일정을 저장하는 테이블이다.

- `id`: 일정 ID
- `lecture_id`: 연결된 강의 ID
- `title`: 일정 제목
- `description`: 일정 설명
- `schedule_date`: 일정 날짜
- `schedule_time`: 일정 시간
- `type`: 일정 유형
- `is_done`: 완료 여부

### 4.3 notes 테이블

초기 구현과의 호환성을 위해 남겨둔 테이블이다. 현재 UI의 직접 필기 기능은 `lectures.summary`를 수정하는 방식으로 통합되어 있다.

## 5. User Interface

LectureMate의 UI는 JavaFX와 별도 CSS 파일을 사용하여 구현했다. 전체 스타일은 흰색, 연회색, 파란색 포인트 컬러를 중심으로 구성했으며, 카드형 UI와 둥근 모서리를 사용하여 학습 도구에 어울리는 깔끔한 화면을 목표로 했다.

화면은 다음과 같은 탭 구조를 가진다.

- 음성 변환
- 강의 목록
- 일정 관리
- 질문하기

상단에는 앱 로고와 작업 상태 표시 영역이 있다. 시간이 오래 걸리는 작업인 음성 변환, 요약 생성, AI 질문 처리 중에는 ProgressIndicator와 상태 메시지를 표시하여 사용자가 현재 작업 상태를 알 수 있도록 했다.

## 7. Error Handling and Fallback

LectureMate는 외부 API 호출 실패 상황을 고려하여 구현했다. API Key가 없거나, API 호출이 실패하거나, 사용량 한도 문제가 발생해도 앱이 종료되지 않는다. 대신 샘플 원문, 샘플 요약, 샘플 답변을 표시하여 기능 흐름을 계속 시연할 수 있도록 했다.

특히 OpenAI API Key는 코드에 직접 작성하지 않고 환경 변수 `OPENAI_API_KEY`에서 읽는다. 이를 통해 GitHub에 소스 코드를 업로드할 때 API Key가 노출되지 않도록 했다.

## 10. Conclusion

LectureMate는 강의 녹음 파일을 바탕으로 원문 텍스트 변환, 자동 요약, 정리 노트 편집, 일정 관리, AI 질문 기능을 제공하는 학습 보조 데스크톱 애플리케이션이다. JavaFX를 이용하여 사용자 인터페이스를 구성했고, SQLite를 사용해 강의와 일정 데이터를 로컬에 저장했다. 또한 OpenAI API 연동 구조를 구현하여 실제 음성 인식과 요약, 질문 답변 기능을 사용할 수 있도록 했다.

이번 프로젝트를 통해 JavaFX 기반 데스크톱 UI 설계, SQLite 데이터베이스 연동, DAO 패턴, 외부 API 호출, 비동기 작업 처리, 예외 처리와 fallback 설계 등을 종합적으로 구현할 수 있었다. LectureMate는 강의 내용을 반복해서 정리해야 하는 학습자에게 유용한 도구가 될 수 있다.
